Architecture Lab

8 Design Labs · mit Lösungsskizze
LAB-01

Landing Zone Blueprint

Management Groups, Subscriptions, RBAC, Policy, Tags, Monitoring und Security Baseline als Zielbild.

Lösungsskizze

MG-Hierarchie: Root → Platform (Identity, Management, Connectivity) + Landing Zones (Corp, Online) + Sandbox + Decommissioned.

Policy-Initiative: Allowed Locations, Require Tags, Deny Public IP auf NICs, DeployIfNotExists für Diagnostics → zentraler Workspace.

RBAC: Entra PIM für privilegierte Rollen, Custom Roles nach Least Privilege, Subscription-Owner nur fürs Platform-Team.

Tags & Monitoring: CostCenter/Env/Owner via „Inherit from subscription", ein Log Analytics Workspace in der Management-Subscription.

governancepolicyrbac
LAB-02

Hybrid Network Design

Hub-Spoke, Private Link, DNS, VPN/ExpressRoute, Front Door und App Gateway sauber abgrenzen.

Lösungsskizze

Topologie: Hub-Spoke. Hub = Azure Firewall + VPN/ER-Gateway + Bastion + DNS Private Resolver. Spokes peeren nur zum Hub (nicht transitiv → UDR auf Firewall).

On-Prem: ExpressRoute primär, Site-to-Site VPN als Failover.

PaaS privat: Private Endpoints + Private DNS Zones zentral im Hub, per VNet-Link auf alle Spokes.

Ingress: Front Door (global, Anycast, WAF Edge) → App Gateway/WAF (regional, Layer-7).

networkprivate-linkdns
LAB-03

BCDR Runbook

RTO/RPO, Backup, Site Recovery, Zones, Region Pair und Wiederherstellungswege in ein Runbook bringen.

Lösungsskizze

Tiering: Mission-critical (RPO≈0 / RTO<1h) → Zones + SQL Auto-Failover Group. Standard (RPO 1h / RTO 4h) → ASR + Geo-Backup. Dev → nur Backup.

VM-DR: Azure Site Recovery in die Paired Region, Recovery Plans für geordnetes Failover.

Daten: SQL Auto-Failover Group (ein Listener), Storage GZRS, Cross Region Restore im Vault aktivieren.

Test: Halbjährlicher Failover-Test, dokumentierte Runbooks, Restore tatsächlich verifizieren.

bcdrrto/rpoasr
LAB-04

Migration Assessment

Rehost, Replatform, Refactor, Azure Migrate, DMS, Dependencies und Risiken in einem Assessment.

Lösungsskizze

Discovery/Assessment: Azure Migrate Appliance erfasst Inventar, Abhängigkeiten, Sizing und Kosten — vor jeder Migration.

6 R's: Rehost (schnell, 1:1), Replatform (PaaS-Mehrwert), Refactor (Cloud-native), Retire/Retain bewusst entscheiden.

Datenbanken: DMS Online-Modus für minimale Downtime; sehr große Datenmengen offline via Azure Data Box.

Waves: nach Abhängigkeiten gruppieren, Risiko-Register + Rollback-Plan je Welle.

migrationcafdms
LAB-05

Identity & Access Blueprint

Hybrid Identity, Conditional Access, PIM, externe Partner und App-Authentifizierung als Zielbild.

Lösungsskizze

Hybrid Identity: Entra Connect mit Password Hash Sync + Seamless SSO. PTA nur, wenn On-Prem-Passwortrichtlinie zwingend live geprüft werden muss.

Conditional Access: Baseline = MFA für alle, risikobasiert via Identity Protection, Gerätekonformität für sensible Apps.

Privileged Access: PIM mit Just-in-Time + Approval + MFA; Break-Glass-Konten ausgenommen und überwacht.

Extern & Apps: B2B-Gäste für Partner, Access Reviews quartalsweise, Managed Identities statt Secrets im Code.

identityconditional-accesspim
LAB-06

Cost Optimization Review

Visibility, Compute-, Storage- und PaaS-Kosten sowie Governance-Guardrails für ein FinOps-Zielbild.

Lösungsskizze

Visibility: Cost Management + Budgets + Anomaly Alerts, durchgängiges Tagging (CostCenter) für Showback.

Compute: Right-Sizing via Advisor, Reserved/Savings Plan für stabile Last, Spot für Batch, Azure Hybrid Benefit für eigene Lizenzen.

Storage: Access Tiers + Lifecycle (Hot→Cool→Archive), Redundanz passend wählen (nicht über-replizieren).

PaaS & Governance: Autoscale, Dev/Test-Auto-Shutdown, Elastic Pool für viele DBs, Policy gegen teure SKUs.

costfinopsreserved
LAB-07

Zero-Trust Security Baseline

Identität als Perimeter, Netzwerk-Härtung, Datenschutz und Detection/Response in einem Sicherheits-Zielbild.

Lösungsskizze

Identität = Perimeter: MFA überall, Conditional Access, Least-Privilege-RBAC, PIM für Admins.

Netzwerk: Private Endpoints, Deny Public Access, Azure Firewall + NSGs, kein offenes RDP/SSH (Bastion + JIT).

Daten: Key Vault mit RBAC, Customer-Managed Keys wo gefordert, TDE, Immutable Storage für Compliance.

Detect/Respond: Defender for Cloud (Secure Score) + Sentinel (SIEM/SOAR) + Diagnostic Settings → zentraler Workspace.

securityzero-trustdefender
LAB-08

Data Platform Design

Relationale und NoSQL-Stores, Analytics, Caching und Daten-DR für eine moderne Datenplattform.

Lösungsskizze

Relational: Azure SQL DB (PaaS) für neue Apps; Managed Instance bei SQL-Server-Kompatibilität/Lift-and-Shift.

NoSQL global: Cosmos DB für niedrige Latenz & Multi-Region-Write; Partition Key mit hoher Kardinalität gegen Hot Partitions.

Skalierung & Analytics: Elastic Pool für viele DBs mit ungleichen Peaks; Synapse (SQL-DWH) bzw. Databricks (Spark/ML); Redis als Cache.

Daten-DR: Auto-Failover Group / Active Geo-Replication, Point-in-Time Restore, georedundante Backups.
